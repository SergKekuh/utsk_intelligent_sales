  • Соответствует требованию ТЗ: «Вести журнал автоматических изменений статусов (кто/когда/какой алгоритм изменил статус)»
-- Пример запроса для аудита истории клиента
SELECT
    c.name,
    old_sr.status_name AS old_status,
    new_sr.status_name AS new_status,
    scl.changed_by,
    scl.changed_at
FROM status_change_log scl
JOIN clients c ON scl.client_code = c.code
LEFT JOIN status_rules old_sr ON scl.old_status_id = old_sr.id
LEFT JOIN status_rules new_sr ON scl.new_status_id = new_sr.id
WHERE scl.client_code = '36'
ORDER BY scl.changed_at DESC;

🔮 БЛОК 5: ВИРТУАЛЬНЫЕ ОБЪЕКТЫ (VIEWS)

15. v_smart_recommendations / v_smart_recommendations_v2 — Движок Рекомендаций

VIEW 🎯 Бизнес-задача: Генерация персональных коммерческих предложений для каждого клиента. Соответствует ТЗ п. 2.3 «Генерация коммерческих предложений».

📥 Источники данных и приоритеты:

ПриоритетИсточникТаблицыМетка в КПЛогика
1История покупокsales_lines, documents«Часто покупаете»Товары, которые клиент покупал 2+ раза
2Новинки по направлениюproducts, clients«Новинка в вашем сегменте»Новые товары из направления клиента
3Сопутствующие товарыproduct_cross_sells«С этим обычно берут»Жесткие кросс-ссылки
4Цифровой следwebsite_behavior_log«Вы недавно интересовались»Товары/категории, просмотренные за 7 дней

⚙️ Алгоритм работы:

  1. Объединяет данные из всех 4 источников
  2. Фильтрует товары по in_stock_balance > 0 (только в наличии)
  3. Исключает товары из manager_rejections_log (отвергнутые за последние 30 дней)
  4. Исключает товары с is_blocked = TRUE в product_scoring
  5. Сортирует по приоритету источника, затем по весу в product_scoring.current_weight
  6. Возвращает не более 5 товаров (соответствует ТЗ п. 2.3)

⚙️ Сортировка по умолчанию:

  1. Сначала клиенты, требующие опроса (requires_survey = TRUE)
  2. Затем по приоритету статуса (status_rules.priority — чем меньше, тем важнее)
  3. Затем по дате последней покупки (сначала недавние)
-- Показать клиентов, с которыми нужно работать сегодня
SELECT code, name, current_status, days_since_last, requires_survey
FROM v_manager_dashboard
WHERE requires_survey = TRUE
   OR days_since_last > 90
   OR current_status IN ('Спящие', 'Ушедшие')
ORDER BY requires_survey DESC, days_since_last DESC;
